# pipeline
Si lo que quieres es un README técnico centrado exclusivamente en el funcionamiento y la configuración del Pipeline de CI/CD, aquí tienes una versión directa y profesional para dejar en la raíz de tu repo:🚀 CI/CD Pipeline: GitHub Actions to Ubuntu VPSEste repositorio cuenta con un flujo de Despliegue Continuo (CD) automatizado. Cada vez que se realiza un push a la rama main, el servidor se actualiza automáticamente utilizando Docker.🛠️ Arquitectura del PipelineTrigger: git push origin main.Runner: GitHub Actions (Ubuntu Latest).Action: Conexión segura vía SSH al VPS.Deploy: - git pull de los últimos cambios.Re-build de las imágenes de Docker.Reinicio de contenedores con docker-compose.🔐 Configuración de Secrets (GitHub)Para que el pipeline funcione, deben estar configurados los siguientes Actions Secrets en el repositorio:SecretDescripciónSSH_HOSTDirección IP pública de tu Ubuntu Server.SSH_USERUsuario con permisos de Docker (ej. root o ubuntu).SSH_KEYContenido de la Clave Privada (id_rsa) con acceso al servidor.📂 Estructura del WorkflowEl archivo de configuración se encuentra en:.github/workflows/deploy.ymlYAML# Resumen del comando ejecutado en el servidor:
cd /ruta/del/proyecto
git pull origin main
docker compose down
docker compose up -d --build
📝 Comandos de Mantenimiento en ServidorSi el pipeline falla, puedes verificar el estado manualmente en el VPS con estos comandos:Ver logs del despliegue: docker compose logs -fVerificar contenedores activos: docker psLimpiar imágenes antiguas: docker image prune -f (Recomendado para ahorrar espacio en el VPS).Nota: Asegúrate de que el usuario definido en SSH_USER pertenezca al grupo docker para evitar errores de permisos en el despliegue automático>
